## LINE Bot 智慧名片管家功能模組規劃與分層設計

本文件旨在整理 LINE Bot 智慧名片管家的現有功能模組，並針對未來提升使用者介面 (UI) 及整合 Rich Menu 的需求，提供分類與分層設計建議。

### 1. 目前所有的功能模組整理

根據現有文件 (`CLAUDE.md` 和 `README.md`)，將功能歸納為以下核心模組：

#### 1.1 名片辨識與儲存 (Card Recognition & Storage)
*   **單張名片辨識**：用戶傳送單張名片圖片，Bot 自動進行 OCR 辨識並結構化儲存。
*   **批量名片上傳**：用戶啟動批量模式 (`新增` 指令)，傳送多張圖片，Bot 進行排程處理 (`完成` 指令)。
*   **名片去重**：系統自動檢查名片重複性 (依 Email)，並提供手動清理重複名片的功能 (`remove` 指令)。

#### 1.2 名片管理與查詢 (Card Management & Query)
*   **智慧搜尋**：用戶輸入任意文字即可進行名片內容的智慧搜尋。
*   **名片編輯**：針對辨識結果或已有名片進行欄位修改 (`editing_field` 狀態)。
*   **備忘錄管理**：為名片添加或編輯備忘錄 (`adding_memo` 狀態)。
*   **標籤系統**：為名片套用或管理標籤 (`adding_tag` 狀態)，支援依標籤篩選 (規劃中)。
*   **vCard 匯出**：生成 QR Code 供用戶掃描，將名片資訊一鍵匯入手機通訊錄。

#### 1.3 團隊協作與組織管理 (Team Collaboration & Organization Management)
*   **團隊資訊查詢**：查看當前團隊名稱與個人身份 (`團隊` / `team` 指令)。
*   **團隊成員管理**：查看團隊成員列表 (`成員` / `members` 指令)。
*   **邀請機制**：管理員生成邀請碼 (`邀請` / `invite` 指令)，新成員透過邀請碼加入團隊 (`加入 <邀請碼>` 指令)。
*   **團隊名稱設定**：管理員修改團隊名稱 (`設定團隊名稱 <名稱>` 指令)。
*   **共享名片庫**：所有團隊成員共享同一個名片資料庫。
*   **角色權限**：區分管理員與一般成員的權限 (例如刪除名片)。

#### 1.4 資料匯出與同步 (Data Export & Synchronization)
*   **CSV 匯出**：將名片資料匯出為 CSV 檔案，並透過 Email 寄送 (`匯出` / `export` 指令)。
*   **Google Sheets 同步**：名片資料自動同步到指定的 Google Sheet。

#### 1.5 系統輔助功能 (System Utilities)
*   **用戶組織初始化**：確保每位用戶都有所屬組織 (個人或團隊)。
*   **錯誤監控與回報**：(規劃中) 追蹤 OCR 成功率、Firebase 錯誤率，並提供用戶回報問題的機制。

### 2. 分類與分層設計建議

為了提升使用者 UI 並為 Rich Menu 做準備，建議將功能分為以下幾個主要類別，並在每個類別下提供更細緻的選項。這將有助於用戶快速找到所需功能，避免指令混亂。

**設計原則：**
*   **用戶導向**：從用戶「想做什麼」的角度出發。
*   **精簡主選單**：Rich Menu 的主選單按鈕不宜過多，3-4 個最佳。
*   **層次分明**：主選單點擊後可透過 Quick Reply 或切換 Rich Menu 進入次級選單。
*   **指令與 UI 結合**：將現有文字指令對應到 UI 按鈕。

#### 建議的分類與分層結構：

**主選單 (Rich Menu Top Level - 3~4 個主要按鈕)**
1.  **名片操作 (Card Actions)**
2.  **團隊功能 (Team Features)**
3.  **資料與設定 (Data & Settings)**
4.  **幫助 (Help)** (可選，或將常見問題整合到其他選單)

#### 詳細功能分層：

**A. 名片操作 (Card Actions)**
*   **目標**：讓用戶能快速新增、查找、管理自己的名片。
*   **Rich Menu 按鈕**：點擊後可彈出 Quick Reply 或進入子選單。
*   **子選單/Quick Reply 建議**：
    *   **新增名片**
        *   `傳送名片圖片` (直接上傳圖片觸發)
        *   `批量上傳` (對應 `新增` 指令，啟動批量模式)
        *   `完成批量` (對應 `完成` 指令，結束批量上傳)
    *   **搜尋名片**
        *   `智慧搜尋` (提示用戶直接輸入關鍵字)
        *   `依標籤篩選` (未來功能，可列出常用標籤供選擇)
    *   **管理名片**
        *   `我的名片列表` (可能導向一個 Flex Message 列表，可選擇編輯、備註、標籤、匯出 vCard)
        *   `清理重複名片` (對應 `remove` 指令)

**B. 團隊功能 (Team Features)**
*   **目標**：提供團隊成員協作與管理組織的功能。
*   **Rich Menu 按鈕**：點擊後可彈出 Quick Reply 或進入子選單。
*   **子選單/Quick Reply 建議**：
    *   **團隊資訊**
        *   `查看團隊資訊` (對應 `團隊` / `team` 指令)
        *   `設定團隊名稱` (對應 `設定團隊名稱 <名稱>` 指令，僅管理員可見)
    *   **成員管理**
        *   `查看成員列表` (對應 `成員` / `members` 指令)
        *   `邀請新成員` (對應 `邀請` / `invite` 指令，僅管理員可見)
        *   `加入團隊` (對應 `加入 <邀請碼>` 指令，提供輸入邀請碼的介面)

**C. 資料與設定 (Data & Settings)**
*   **目標**：處理資料的進出與系統相關設定。
*   **Rich Menu 按鈕**：點擊後可彈出 Quick Reply 或進入子選單。
*   **子選單/Quick Reply 建議**：
    *   **匯出資料**
        *   `匯出 CSV` (對應 `匯出` / `export` 指令)
    *   **同步設定**
        *   `Google Sheets 同步狀態` (顯示是否啟用及狀態)
    *   **個人設定** (未來可擴展)
        *   `通知設定`
        *   `帳戶管理`

**D. 幫助 (Help)** (可選，若 Rich Menu 空間有限，可將此功能整合到其他選單或透過文字指令觸發)
*   **目標**：提供用戶支援與指引。
*   **子選單/Quick Reply 建議**：
    *   `使用說明` (連結到外部文件或 Flex Message 介紹)
    *   `常見問題` (FAQ)
    *   `問題回報` (引導用戶輸入問題，例如 `問題回報 [內容]`)

#### 實作考量：
*   **Rich Menu 設計**：設計多個 Rich Menu，例如一個主選單，點擊主選單按鈕後，切換到對應的子選單 Rich Menu。或者，主選單按鈕觸發 Quick Reply，提供次級選項。
*   **上下文感知**：某些操作 (如編輯名片) 應在用戶查看特定名片時，透過 Flex Message 的按鈕直接觸發，而非透過通用選單。
*   **權限控制**：在 Rich Menu 或 Quick Reply 中，針對管理員專屬的功能，可以只對管理員顯示，或顯示後點擊提示權限不足。
*   **文字指令保留**：即使有了 Rich Menu，仍應保留文字指令作為快速操作或備用方案。
